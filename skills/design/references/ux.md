# UX

## Scanning behavior

- users scan before they read deeply
- headings, spacing, grouping, and emphasis should support fast orientation

## UI hierarchy

- make the primary task obvious
- keep supporting actions visible but secondary
- avoid crowded control surfaces

## Responsiveness

- layout and hierarchy must survive screen-size changes
- do not just shrink desktop designs
- prioritize task completion on smaller screens

## Usability principles

- clarity over novelty
- consistent interactions over one-off creativity
- visible states for loading, empty, error, and success
