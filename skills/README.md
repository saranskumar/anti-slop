# Active Skills

This directory is the active Antigravity developer/designer skill surface.

## Routing model
- Use `product-intake-processor` only when the request is broad and needs to be converted into implementation requirements.
- For all other requests, prefer the narrowest skill that matches the actual engineering or design task.

## Core families
- architecture and environment planning
- backend services, APIs, workers, and databases
- auth, security, trust boundaries, and secrets hygiene
- Python services, CLIs, automation, packages, and workers
- TypeScript and JavaScript backend design
- Supabase and Firebase implementation planning
- broad visual design and implementation-relevant UX design
- UI/UX, frontend planning, and React component generation
- tooling, configuration, deployment, review, testing, and observability
