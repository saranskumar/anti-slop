---
name: monetization-planner
description: Use when the team needs pricing, packaging, free-tier, trial, usage metering, or business-model guidance for a software product. Trigger on requests to design how the product should make money or control value access.
---

# Goal
Pick a monetization model that fits value delivery and operating costs.

# When to Use
- The MVP is becoming launchable.
- AI, storage, or service costs make packaging important.
- Plan structure or paywall timing is undecided.

# Instructions
1. Identify value metric and cost drivers.
2. Compare seat, subscription, usage, transaction, or hybrid approaches.
3. Recommend plan boundaries and free access logic.
4. Explain how pricing aligns with user outcomes.
5. Flag risks like margin collapse or packaging confusion.

# Constraints
- Do not invent enterprise pricing complexity for an unvalidated product.
- Avoid free tiers that subsidize heavy cost.
- Keep the first pricing model explainable.

# Output Format
- Monetization recommendation
- Packaging structure
- Free/trial logic
- Risks and future evolution

# Examples
- "How should we monetize this AI SaaS?"
- "Seat or usage pricing for a devtool?"
