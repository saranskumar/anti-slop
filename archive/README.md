# Archive

## Why these assets were archived
- ideation-heavy or startup-oriented
- duplicated by narrower specialist skills
- too broad for reliable auto-selection
- outdated, placeholder-like, or low-signal for a developer/designer operating system

## Active surfaces
- active skills remain in `../skills/`
- active workflows remain in `../global_workflows/`
- active rules remain in `../rules/`
