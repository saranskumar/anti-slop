# Design System Rule

## Style

- premium SaaS
- light mode first
- minimal but powerful

## Principles

- strong hierarchy
- clean spacing
- modular layout
- no clutter

## UI Patterns

- sidebar navigation
- cards and panels
- tabs for segmentation
- clear CTAs

## Interaction

- actions must be obvious
- no hidden complexity
- selection must be easy

## Design Language

- Notion-like clarity
- Modern SaaS administration interface
- Strong information architecture
- Minimal visual chaos

## When Designing a Screen

Always provide:
1. page goal
2. layout structure
3. component hierarchy
4. key user actions
5. edge states

## Avoid

- AI-looking UI
- overuse of gradients
- visual noise
- generic AI-dashboard aesthetics
- Dribbble-style fluff
