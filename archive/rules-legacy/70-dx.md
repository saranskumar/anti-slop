# DX Layer Rule

## Package manager
- ALWAYS use pnpm. Never npm unless pnpm explicitly fails.
- Monorepo support via pnpm workspaces when needed.

## Code quality toolchain
- ESLint: lint on save and pre-commit.
- Prettier: format on save. Never argue about formatting.
- Husky: run lint-staged before every commit.

## Setup assumption
Assume these are already configured. Do NOT suggest removing or replacing them.
When scaffolding new packages or scripts, use pnpm syntax only.
