# Anti-Patterns

- pretending a web-only skill is runtime-agnostic
- mixing trusted and untrusted code paths
- duplicating business logic across UI and server layers
- building broad "assistant" skills instead of focused capabilities
- using ideation workflows when the real need is implementation planning
- storing secrets in source code or frontend bundles
- designing data models without ownership and access rules
- creating queues, microservices, or serverless functions without operational need
